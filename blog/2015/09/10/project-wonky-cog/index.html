Project Wonky Cog

Sep 10th, 2015 1:27 pm

Time to start a new project, it’s a mash-up aimed directly at me.

Called Wonky Cog because as the only user I shan’t be to bothered if things get out of balance or ugly. Hopefully this will give me more creative freedom, a place to try out new ideas. I want somewhere I can add and remove web components, and the rest, easily. The site will be public facing but password protected.

So what framework/s to use for this single page app (SAP) that will use Firebase as a back-end. The obvious choice it seemed to me was the Ember Framework but while looking for tutorials I did discover the Aurelia Framework. Aurelia is still in alpha, going beta in four weeks. What I like most is it’s use of ES6/7 scripting via the Babel transpiler, btw the other build tools are great too. I won’t go into much detail here other than to say it’s easy to get started, the code is very readable and the build system is easily understood.

In conclusion, Aurelia (with Bootstrap 4) is the way I will go. Wonky Cog has already been useful, it’s given me the opportunity to learn ES2015 and beyond.
